1. 트랜젝션.
= 데이터베이스에서 트랜잭션(Transaction)은 데이터 처리의 한 단위.
길게 서술하자면,
오라클에서 발생하는 여러 개의 SQL 명령문들을 하나의 논리적인 작업 단위로 처리하는데 이를 트랜잭션이라고 한다.

- 하나의 트랜잭션은 All-OR-Nothing 방식으로 처리된다.
-   여러 개의 명령어의 집합이 정상적으로 처리되면 정상 종료하도록 하고 
    	여러 개의 명령어 중에서 하나의 명령어라도 잘못되었다면 전체를 취소해버립니다.
-   데이터베이스에서 작업의 단위로 트랜잭션이란 개념을 도입한 이유는 
	데이터의 일관성을 유지하면서 안정적으로 데이터를 복구시키기 위함이다. 
2. 트랜젝션의 명령어들.
COMMIT
SAVEPOINT
ROLLBACK

DML 에서 수행한 작업들은 실제로 오라클 데이터베이스에 들어가는것이 아니라서 자바나 다른 프로그램에서
sql 연동해서 작업할때 안되는경우중 하나가 바로 COMMIT 작업을 안해서 그런것인데, 
COMMIT 작업은
정상적으로 처리하겠다고 확정하는 명령어로 트랜잭션의 처리 과정을 데이터베이스에 모두 반영하고 모두 영구 저장하는것이다.

ROLLBACK 작업은 트랜잭션 처리 과정중에서 잘못넣은 값이나 다시 되돌려야할때 ROLLBACK 을 쓰게 된다.
그러면 트랜잭션으로 인한 하나의 묶음처리가 시작되기전의 상태로 되돌린다.
그러나 COMMIT을 하고 아무리 ROLLBACK을 해도 이미 영구저장된 값이라 되돌릴수가없다.

SAVEPOINT 는 간단히 말하자면, COMMIT 를 하고 다음 작업문에서 SAVEPOINT 를 지정하면 그 지정한 지점으로 다시 돌아간다.
문법은 SAVEPOINT "c1" ; OR  ROLLBACK TO "c1" ;
